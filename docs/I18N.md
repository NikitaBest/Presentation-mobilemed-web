# Локализация (ru / en)

## UI

- **Первый вход**: экран выбора языка (`LanguageSelectPage`) — пока в `localStorage` нет `mm_locale_chosen` и сохранённой `mm_app_locale`. После выбора — главная (Welcome).
- Выбор языка позже: стартовый экран (Welcome) и блок **Настройки** на экране результатов.
- Строки интерфейса: `src/i18n/messages.js` — ключи `welcome.*`, `results.*`, `lang.*`, `common.*`, `app.*`, `userData.*`, `instruction.*`, `scan.*` (включая `scan.iv.*` / `scan.pill.*`), `healthScore.*`, `welcome.sheet*`, `metricGrid.*`, `metricSheet.*`.
- Экраны: LanguageSelect (первый вход), Welcome, UserData, подготовка к скану (Instruction), Scan, Results; общие подсказки в `App.jsx` (Suspense, ошибки auth/профиля). Авторизация (`ensureAuthSession`) запускается только после выбора языка.
- Демо-данные главной: `src/pages/welcomeDemoData.js` (шаги и карточки превью).

## Бэкенд (см. `docs/API.md`)

- **`POST /auth/login`**: в теле передаётся **`locale`: `"ru"` или `"en"`** — попадает в JWT (claim Locale).
- **`apiFetch`**: на каждый запрос добавляется **`Accept-Language: ru|en`** (если заголовок не задан вручную).
- Смена языка при уже выданном JWT: повторный **login** с тем же `id` пользователя и новым `locale` (см. `reauthenticateWithCurrentLocale` в `src/api/auth.js`).

## Хранение

- Ключ **`mm_app_locale`** в `localStorage` — выбранный язык (`ru` / `en`).
- Ключ **`mm_locale_chosen`** (`1`) — пользователь прошёл экран выбора языка (или сменил язык в Welcome/Results).
- До первого выбора для подписей в `I18nProvider` временно берётся `navigator.language`.
