Интеграция SDK
После получения BiosenseSignal_Web_Sample_X.X.X.zipфайла вы готовы добавить Web SDK в свое приложение.

Выполните следующие шаги, чтобы интегрировать SDK в ваше приложение.

1. Добавьте пакет BiosenseSignal SDK в свой проект.
1.1. Распакуйте BiosenseSignal_Web_Sample_X.X.X.tgzфайл.

1.2. Скопируйте biosensesignal-web-sdk-vX.X.X-X.tgzфайл в папку вашего проекта.

1.3. Используйте npmдля установки:


npm install biosensesignal-web-sdk-vX.X.X-X.tgz
Или yarn:


yarn install biosensesignal-web-sdk-vX.X.X-X.tgz
1.4. Добавьте следующее в webpack.config.jsплагины для ваших файлов:


    plugins: [
      new CopyPlugin({
        patterns: [
          {
            from: path.resolve(paths.node_modules, '@biosensesignal/web-sdk/dist'),
            to: path.resolve(paths.build),
            globOptions: {
              ignore: ["**/main.*"]
            }
          },
        ],
      }),
    ]

Создание сессии измерений
Сессия — это интерфейс для проведения измерений жизненно важных показателей.

Одновременно можно создать только одну сессию. Перед созданием новой сессии необходимо завершить предыдущую.
Сеанс предназначен для одного пользователя. При измерении жизненно важных показателей другого пользователя необходимо создать новый сеанс. См. раздел «Информация о пользователе» .
Следующий код можно использовать для создания сессии с соответствующими параметрами:

Завершение сессии
Рекомендуется завершать сессию всякий раз, когда экран измерения перестаёт отображаться. Этот terminate()метод обеспечивает безопасный и структурированный способ завершения активной сессии. Он гарантирует надлежащее освобождение всех выделенных ресурсов, позволяя при этом немедленно продолжить выполнение.

Описание

Завершает активную сессию и освобождает ресурсы. Эта функция инициирует процесс завершения, но не блокирует выполнение, пока ожидает его завершения.

Использование


Машинопись
Машинопись
session.terminate();
console.log("Session termination initiated.");
Поведение

Если сессия уже завершена, функция ничего не делает.
Если сессия активна, процесс завершения запускается асинхронно.
Неблокирующее поведение : функция возвращает управление немедленно, даже если очистка ресурсов еще продолжается.
Ожидаемое поведение разработчиков

✅ Позвоните, terminate()когда сеанс больше не нужен.

✅ Не следует предполагать немедленное завершение ; если логика зависит от завершения сессии, реализуйте необходимую логику ожидания, прежде чем продолжить.

✅ Обрабатывать все предупреждения и записи в журнале для мониторинга процесса завершения работы.

❌ Не пытайтесь перезапустить прерванную сессию. Вместо этого необходимо создать новую сессию.

Пример использования


Машинопись
Машинопись
session.terminate();
console.log("Session termination initiated.");
Перед продолжением убедитесь в завершении процесса. Если последующие действия зависят от завершения сессии, отслеживайте OnStateChangeобратный вызов и убедитесь, что сессия завершена, прежде чем продолжить:


Машинопись
Машинопись
import { 
    SessionState, 
} from '@biosensesignal/web-sdk';

const onStateChange = useCallback((state: SessionState) => {
    switch (state) {            
        case SessionState.TERMINATED:
            console.log("Session is terminated");
            break;
    }
}, []);
Жизненный цикл сессии: процесс завершения

Вызов разработчика terminate()→ Сессия переходит в фазу завершения.
Начинается внутренняя очистка (асинхронно) → Освобождаются ресурсы, такие как камера и процессоры.
Состояние сессии обновляется после TERMINATEDзавершения очистки.
Дальнейшие вызовы API, зависящие от активной сессии, завершатся неудачей.
Часто задаваемые вопросы

❓ Как я могу узнать, когда процедура прерывания завершена?

Поскольку terminate()это неблокирующий процесс, не следует предполагать немедленное завершение. Вместо этого: отслеживайте состояние сессии до достижения соответствующего SessionState.TERMINATEDсостояния. Прослушивайте системные журналы для подтверждения.

❓ Что произойдет, если я позвоню terminate()дважды?

Обращение terminate()к уже завершенной сессии не имеет никакого эффекта.

❓ terminate()Вызовет ошибки?

Нет, terminate()обработка ошибок происходит внутри системы. Если требуется обработка ошибок, отслеживается состояние сессии.

Заключение

Используйте terminate()для корректного завершения сессии.
Ожидается неблокирующее поведение — завершение работы происходит в фоновом режиме.
При необходимости проверьте состояние сессии до и после ее завершения.

Состояние сессии
Сеанс измерений всегда находится в «состоянии». Сеанс переходит между возможными состояниями либо в результате выполнения действия API, вызываемого приложением, либо с помощью внутренней логики, предназначенной для подготовки сеанса к выполнению измерений. Диаграмма состояний сеанса представлена ​​на рисунке ниже.

Штаты сессии
Диаграмма состояний

В таблице ниже приведено описание каждого состояния сессии:

Название штата	Определение штата
ИНИЦИАЛИЗАЦИЯ	Сессия находится в исходном состоянии и выполняет действия по инициализации. Пожалуйста, подождите, пока не получите сообщение о том, что сессия находится в состоянии АКТИВНОСТИ , прежде чем начинать измерение жизненно важных показателей или вызывать какие-либо API сессии.
АКТИВНЫЙ	Сеанс готов к запуску. Приложение может отображать предварительный просмотр с камеры, если используются измерения лица. Подробные инструкции см. на странице «Создание предварительного просмотра» .

ОСТАНОВКА	Сеанс остановлен, и результаты измерений обрабатываются. Информацию об обработке окончательных результатов см. на странице « Жизненно важные показатели» .
Уволен	Сеанс корректно завершился, и теперь можно начать новый сеанс.
Переходы состояний сессии
В таблице ниже описаны действия, вызывающие переход между состояниями:

Состояние	Следующий штат	Курок
ИНИЦИАЛИЗАЦИЯ	АКТИВНЫЙ	После завершения всех действий по инициализации сессия переходит в состояние ACTIVE .
АКТИВНЫЙ	ИЗМЕРЕНИЕ	Вызов этого start()метода приводит к переходу сессии в состояние MEASURING .

Состояние STOPPING — это «переходное состояние», которое автоматически завершается через короткий промежуток времени. Не вызывайте никакие методы сессии, пока сессия находится в переходном состоянии.

Приведённый ниже код — это простой пример обработки обновлений при переходе между сессиями с помощью OnStateChangeфункции обратного вызова:


Машинопись
Машинопись
import { 
    SessionState, 
} from '@biosensesignal/web-sdk';

const onStateChange = useCallback((state: SessionState) => {
    switch (state) {
        case SessionState.INIT:
            console.log("Session is initializing and NOT ready");
            break;
        case SessionState.ACTIVE:
            console.log("Session is ready to start measuring");
            break;
        case SessionState.MEASURING:
            console.log("Session is measuring vital signs");
            break;
        case SessionState.STOPPING:
            console.log("Session is stopping the measuring of vital signs");
            break;
        case SessionState.TERMINATED:
            console.log("Session is terminated");
            break;
    }
}, []);

Информация о пользователе
Для расчета показателей риска атеросклеротических сердечно-сосудистых заболеваний (АССЗ) и « сердечного возраста» SDK требует получения информации от пользователя , включая данные о пользователе, проводившем измерение. Если эта информация не предоставлена ​​SDK, расчет этих показателей производиться не будет.

Информация о пользователе включает в себя следующие поля:

Пол (по данным рождения) [НЕ УКАЗАН / МУЖСКОЙ / ЖЕНСКИЙ]
Возраст [лет]
Вес [в килограммах]
Высота [сантиметры]
Статус курения [НЕ УКАЗАНО / КУРИЛЬЩИК / НЕ КУРИЛЬЩИК]

Ориентация устройства
SDK поддерживает настройку ориентации устройства в сеансах измерения лица. Ориентация определяется приложением во время создания сеанса и может быть установлена ​​в соответствии с текущей ориентацией устройства на момент создания сеанса или в соответствии с предпочтительной ориентацией пользовательского интерфейса.

Ориентация определяется как положение основания устройства (также известного как местоположение зарядного порта) относительно текущего поворота устройства. Например, если устройство повернуто так, что его основание находится слева от пользователя, то ориентация определяется как LANDSCAPE_LEFT.

При создании сессии, если не указана конкретная ориентация, допустимой ориентацией считается ориентация устройства на момент вызова команды 'start'.

В следующем примере ориентация устройства следующая LANDSCAPE_LEFT:

Изображения с неправильной ориентацией не будут обрабатываться SDK.
Достоверность изображения
Хотя основная инструкция по проведению измерения проста — достаточно посмотреть в камеру и начать измерение — существует несколько рекомендаций, которым пользователь должен следовать для обеспечения точных результатов измерения. Эти рекомендации перечислены в руководстве по передовым методам проведения измерений .

В процессе измерения SDK помогает пользователю следовать этим рекомендациям. Он проверяет каждое изображение с камеры и обновляет данные ImageValidityпри обнаружении любых отклонений от рекомендаций.

Изображение считается действительным, если SDK не обнаружил никаких нарушений передовых методов проведения измерений . Если изображение не считается действительным, SDK сообщает причину признания изображения недействительным.

Условия, указанные в таблице ниже, сделают изображение непригодным для обработки SDK.

Имя	Значение
ДЕЙСТВИТЕЛЬНЫЙ	Изображение корректно.
INVALID_DEVICE_ORIENTATION	Ориентация устройства в рамках данной сессии не поддерживается.
INVALID_ROI	SDK не может распознать лицо пользователя.
TILTED_HEAD	Лицо пользователя не обращено прямо к камере.
FACE_TOO_FAR	В настоящее время не поддерживается в веб-версии.
НЕРАВНОМЕРНЫЙ СВЕТ	Свет, падающий на лицо пользователя, распределяется неравномерно.

Руководство по измерениям
Для точных измерений лица с помощью SDK BiosenseSignal пользователю необходимо следовать рекомендациям по проверке изображений. SDK помогает пользователям придерживаться рекомендаций по измерениям, указывая исключения в информации о проверке изображений. Подробную информацию о проверке изображений см. на странице «Проверка изображений» .

SDK уведомляет приложение о корректности изображения каждого кадра. Настоятельно рекомендуется запрашивать у пользователя информацию о любых обнаруженных исключениях и инструктировать его следовать передовым методам проведения измерений . Используйте пример кода приложения для реализации уведомлений о корректности изображения.

SDK поддерживает настройку включения строгих рекомендаций по измерению. Этот параметр определяет, будет ли SDK обрабатывать все видеоизображения при обнаружении лица (поведение по умолчанию) или только изображения с подтвержденной достоверностью. В следующем примере строгие рекомендации по измерению установлены на true:

Строгие рекомендации по измерениям установлены на «Нет» (по умолчанию).
Если установлено значение false (поведение SDK по умолчанию), SDK обрабатывает все видеоизображения до тех пор, пока обнаружено лицо (существуют данные изображения в области интереса).

Строгие правила измерения установлены на «Истина».
Если этот параметр установлен в значение true, SDK будет обрабатывать только действительные изображения лиц для обеспечения повышенной точности.

Если последовательность некорректных изображений сохраняется более 0,5 секунды, SDK предупреждает о значительном разрыве. Помимо влияния на точность результатов, эти разрывы могут также вызывать задержки в отображении жизненно важных показателей и снижать уверенность в окончательных результатах.

При третьем возникновении задержки в 0,5 секунды SDK выдает ошибку, останавливая сессию без получения окончательных результатов. Такое поведение статистически повышает точность измерений и побуждает пользователей более эффективно следовать рекомендациям по измерениям.

Оповещения
Оповещения — это сообщения, отправляемые из SDK в приложение при возникновении неисправности. Существует две категории оповещений:

Предупреждение - указывает на незначительную, временную проблему, которая не нарушает текущую работу.
Ошибка — указывает на серьезную проблему, которую не удается устранить с помощью SDK, и приводит к прекращению любых текущих измерений.
Интерфейс оповещения содержит числовой код, обозначающий конкретную проблему.

Причины предупреждений и ошибок могут быть различными — от проблем с точностью до некорректного использования API, проблем с лицензиями или даже ошибок, связанных с устройством.

Получение оповещений
В случае получения оповещения рекомендуется отобразить числовой код оповещения в пользовательском интерфейсе для справки. Если оповещение связано с неправильным использованием SDK или некорректными условиями измерения, и проблема сохраняется, обратитесь в службу поддержки.

Приложение может получать оповещения с помощью интерфейсов обратного вызова OnError и OnWarning .

Ошибки
Ошибка указывает на серьезный инцидент, после которого SDK не может восстановиться. К распространенным причинам ошибок относятся проблемы с сетью, перегрузка процессора, недостаточное освещение и неблагоприятные условия окружающей среды. Подробную информацию о каждой ошибке см. в списке предупреждений .

При возникновении ошибки во время измерения SDK прекратит текущее измерение и не будет отправлять результаты измерений в приложение. Кроме того, состояние сессии перейдет из состояния «ИЗМЕРЕНИЕ» в состояние «ОСТАНОВКА» (см. Состояния сессии ).

Внутренние ошибки
Если во время работы SDK возникает недопустимое внутреннее состояние, приложению будет возвращена внутренняя ошибка. Номер внутренней ошибки может не отображаться на странице списка предупреждений . Как упоминалось ранее, рекомендуется отображать номер ошибки в пользовательском интерфейсе для справки.

Если вы получили сообщение о внутренней ошибке, пожалуйста, сообщите об этом в службу поддержки. Укажите полученный код ошибки и предоставьте подробное описание проблемы.

Маршрутизация сети сервера лицензий
SDK подключается к серверу лицензий по адресу https://licensing-api.biosensesignal.com . Трафик к этому серверу маршрутизируется через сервис Cloudflare. Поскольку Cloudflare недоступен в некоторых странах, для этих регионов доступно специальное обходное решение. Обратитесь в нашу службу поддержки клиентов, если сервер лицензий недоступен на целевых территориях.

Лицензия на проведение сессий
При использовании сессионного лицензирования сервер лицензий предоставляет SDK выделенное количество измерений (или «квоту»), как указано в лицензионном соглашении.

Для работы SDK требуется подключение к интернету, позволяющее обмениваться данными с сервером лицензий для проверки действительности лицензии.

Потребление измерительных приборов
При вызове startметода SDK дает указание серверу лицензий обработать одно измерение и изменяет состояние сессии с ACTIVE на MEASURING . Если на сервере нет доступных измерений, процесс будет прерван, SDK отправит ошибку, и сессия вернется в состояние STOPPING и ACTIVE (см. Состояние сессии ).

Примечание

При использовании сессионной лицензии SDK также передает идентификатор активации приложению . Однако при использовании этого типа лицензии квота на активацию неограничена.

Осталось времени до окончания действия таймера лицензии.
Для поддержки случаев, когда сессия была израсходована из квоты лицензии, но измерение не удалось по какой-либо причине, например, из-за входящего телефонного звонка, приложение может выполнять повторные измерения без использования дополнительных сессий из лицензии. При начале первого измерения запускается таймер «временного интервала сессии». Изначально таймер устанавливается на 9 минут (540 секунд). В течение этого периода приложение может выполнять повторные измерения без использования дополнительных сессий из лицензии.

Примечание

Этот раздел актуален только для лицензий с пользовательской конфигурацией, включающей более одного автономного измерения, как это настроено на сервере лицензий. Для настройки этой конфигурации требуется помощь службы поддержки.

По умолчанию механизм лицензирования настроен на использование 1 измерения из квоты лицензии на сервере при вызове метода session start. Некоторые лицензии настроены на получение дополнительных измерений с сервера и их локальное сохранение в SDK для дальнейшего использования. Это позволяет приложению запускать дополнительные сессии, даже если устройство не имеет доступа к интернету в режиме реального времени.

Следующие параметры указывают на состояние локально сохраненных измерений:

Автономные измерения — общее количество измерений, которые могут быть сохранены локально на устройстве.
Оставшиеся измерения — общее количество измерений, которые уже были загружены с сервера в SDK и могут быть использованы в будущих сессиях.
